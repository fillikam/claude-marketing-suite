# Как работают скиллы Claude

## Что такое скилл

Скилл -- это расширение для Claude Code, которое добавляет специализированную команду. Каждый скилл состоит из двух файлов:

- **`.skill`** -- YAML-конфигурация: имя, описание, триггеры (слэш-команда), метаданные
- **`SKILL.md`** -- подробная инструкция (промпт), которую Claude получает при активации скилла

## Как Claude использует скилл

1. Вы вводите команду, например `/aim audit https://example.com`
2. Claude Code находит соответствующий `.skill` файл по триггеру
3. Claude загружает `SKILL.md` как системную инструкцию
4. Claude выполняет задачу согласно инструкции и генерирует выходной файл

## Кросс-скилл интеграция

Скиллы в Claude Marketing Suite спроектированы для совместной работы. Когда скилл запускается, он проверяет наличие выходных файлов от других скиллов в текущей директории:

- `MARKETING-AUDIT.md` -- данные аудита
- `BRAND-VOICE.md` -- тональность бренда
- `AUDIENCE-RESEARCH.md` -- профиль аудитории
- и другие

Если файлы найдены -- скилл использует их для более точных и персонализированных результатов. Если нет -- работает автономно на основе анализа сайта.

## Структура выходных файлов

Каждый скилл создаёт один Markdown-файл с:
- Структурированными разделами (заголовки H2/H3)
- Таблицами и списками для наглядности
- Actionable-рекомендациями (не общие советы, а конкретные шаги)
- Метаданными (дата, URL, версия скилла)

## Guardrails

Скиллы включают встроенные проверки качества:
- Минимальная длина вывода
- Обязательные разделы
- Проверка на «водянистые» формулировки
- Fallback-протокол при недостатке данных
